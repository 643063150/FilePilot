# FilePilot v1 问题修复计划

> **状态**：进行中

## 问题清单（按优先级排序）

### P0 - 严重问题（必须立即修复）

#### 1. 侧边栏无法点击
- **现状**：侧边栏按钮渲染器未正确初始化，导致点击无响应
- **原因**：`SidebarButton.CreateRenderer()` 中 `bg`, `icon`, `label` 字段未初始化
- **修复方案**：
  - 在 `CreateRenderer()` 中正确初始化所有组件
  - 添加点击事件处理
  - 测试按钮响应

#### 2. 搜索结果列表只显示图标，无文件信息
- **现状**：搜索结果列表项只显示一个图标，没有文件名、路径等信息
- **原因**：`HomePage.createResultItem()` 和 `updateResultItem()` 实现不正确
- **修复方案**：
  - 重写结果项布局：图标 + 文件名 + 路径 + 大小
  - 使用 Card 组件包裹每个结果项
  - 确保数据正确绑定到 UI 组件

#### 3. 界面完全不是 Material Design 3 风格
- **现状**：使用默认 Fyne 组件，无圆角、无阴影、无层次感
- **原因**：自定义 MD3 组件未正确实现，只是定义了结构但未实现渲染逻辑
- **修复方案**：
  - 实现完整的 MD3 组件渲染器（CreateRenderer）
  - 或者：**切换到 Go + H5 方案**，使用 CSS 实现 MD3 风格
  - 推荐切换到 H5 方案，因为 Fyne 自定义组件开发复杂度高

#### 4. 首页交互逻辑不符合浏览器模式
- **现状**：搜索结果和预览混在一起，没有清晰的导航流程
- **目标**：采用浏览器式交互
  - 搜索后点击文件 -> 当前标签页显示预览（覆盖搜索界面）
  - 新标签页 -> 显示首页初始状态
  - 支持返回按钮回到搜索结果
- **修复方案**：
  - 重构首页为多状态视图（初始状态、搜索结果状态、预览状态）
  - 每个状态独立管理
  - 标签页管理器支持状态切换

### P1 - 重要问题

#### 5. 每个页面都有莫名其妙的进度条/滑动条
- **现状**：`container.NewVScroll` 在每个页面都显示滚动条
- **目标**：只在需要时显示滚动条，且样式美观
- **修复方案**：
  - 使用自定义滚动容器，隐藏不必要的滚动条
  - 或者使用 Fyne 的 `container.NewVBox` 配合 `container.NewMax`

#### 6. 文件夹卡片样式错误
- **现状**：普通 Card 组件，无虚线边框，无加号图标
- **目标**："添加文件夹"卡片虚线边框 + 加号图标，强调色 `#4f8cf7`
- **修复方案**：
  - 自定义卡片组件，使用 Canvas 绘制虚线边框
  - 添加加号图标居中显示

#### 7. CPU 利用率低，索引时间远超预期
- **现状**：单线程遍历，无并发处理，28717 个文件约 9 分钟
- **目标**：5000 个文件 <= 5 分钟（SSD）
- **修复方案**：
  - 使用 goroutine 池并发处理
  - 限制最大并发数 = CPU 核心数 x 2
  - 批量插入数据库（每 100 条事务提交）
  - 使用 `filepath.WalkDir` 替代 `filepath.Walk`（性能更好）

### P2 - 一般问题

#### 8. 进度动画不符合规范
- **现状**：使用 Fyne 默认进度条
- **目标**：模态框 + 毛玻璃遮罩 + 环形进度条 + 百分比
- **修复方案**：

  - 使用 Canvas 绘制环形进度条

#### 9. 设置页面缺少功能
- **现状**：基础表单布局
- **目标**：单选按钮、数字输入框、开关、危险按钮
- **修复方案**：按规范重新设计设置项

#### 10. 统计页面缺少图表
- **现状**：文本表格显示
- **目标**：饼图展示文件类型占比
- **修复方案**：使用 Canvas 绘制饼图或使用第三方图表库

## 技术方案决策

### ✅ 已确定方案：Go + H5（Vue + Vuetify）

**技术栈**：
- 后端：Go + Gin（HTTP 框架）
- 前端：Vue 3 + Vuetify 3（Material Design 3 组件库）
- 通信：WebSocket（实时更新）+ REST API
- 打包：`github.com/webview/webview` 或 Electron

**架构设计**：
```
┌─────────────────────────────────────────────┐
│                  Webview 窗口                │
│  ┌─────────────────────────────────────────┐ │
│  │           Vue 3 + Vuetify 3            │ │
│  │         (Material Design 3 UI)         │ │
│  └─────────────────────────────────────────┘ │
│                    ↕ HTTP/WS                 │
│  ┌─────────────────────────────────────────┐ │
│  │           Go Gin Server                │ │
│  │         (业务逻辑 + SQLite)             │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**前端目录结构**：
```
frontend/
├── src/
│   ├── components/      # 通用组件
│   ├── views/           # 页面视图
│   ├── stores/          # Pinia 状态管理
│   ├── router/          # Vue Router
│   ├── plugins/         # Vuetify 配置
│   └── assets/          # 静态资源
├── public/
└── package.json
```

**后端目录结构**：
```
backend/
├── main.go              # 入口
├── handlers/            # HTTP 处理器
├── services/            # 业务逻辑
├── models/              # 数据模型
├── indexer/             # 文件索引器
└── database/            # 数据库操作
```

## 新增问题 (UI/UX) - 2026-07-04

### 问题 11: 界面布局拥挤
**严重程度：** 高
**问题描述：** 界面元素挤在一起，间距不够，视觉层次不清晰。
**修复方案：**
- 增加页面内边距和元素间距
- 优化卡片布局，增加呼吸空间
- 调整侧边栏与主内容区的比例
- 添加适当的分隔线和空白区域

### 问题 12: 添加文件夹只能手动输入路径
**严重程度：** 高
**问题描述：** 添加文件夹时只能手动输入路径，无法通过系统对话框选择文件夹。
**修复方案：**
- 后端添加文件夹选择对话框 API（使用 `github.com/ncruces/zenity`）
- 前端调用系统文件夹选择器
- 保留手动输入作为备选方案

### 问题 13: 未打包为 exe 程序
**严重程度：** 高
**问题描述：** 当前是 Web 应用形式，需要打包为独立的 exe 程序。
**修复方案：**
- 使用 `github.com/webview/webview` 嵌入 Webview
- 将前端资源嵌入到 exe 中
- 使用 `-H windowsgui` 编译隐藏控制台
- 创建 Inno Setup 安装脚本

### 问题 14: 首页缺少浏览器式标签页
**严重程度：** 高
**问题描述：** 点击文件预览后无法返回，缺少浏览器式的标签页和前进/后退导航。
**修复方案：**
- 实现标签页组件（可关闭、可切换）
- 每个标签页独立维护浏览历史
- 支持前进/后退导航
- 双击文件打开新标签页

### 问题 15: 右键菜单缺失
**严重程度：** 中
**问题描述：** 文件列表和文件夹卡片没有右键菜单。
**修复方案：**
- 实现自定义右键菜单组件
- 搜索结果右键：打开、打开所在位置、复制路径
- 文件夹卡片右键：重新索引、移除
- 最近文件右键：打开、从记录中移除
