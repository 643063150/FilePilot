# 问题报告：卸载清理不完整

> 报告时间：2026-07-05
> 问题严重性：高

---

## 问题描述

卸载程序无法完全清理所有数据，导致：
1. 重装后仍保留旧数据（索引、设置、历史记录）
2. 用户无法真正"重新开始"

---

## 根本原因

### 数据存储位置分散

FilePilot 经历了多次技术栈变更，数据存储在多个位置：

| 版本 | 技术栈 | 数据位置 |
|------|--------|----------|
| v1 (早期) | Go + Fyne | `%APPDATA%\fyne\com.filepilot.app` |
| v2 (中期) | Go + Web | `%USERPROFILE%\.filepilot` |
| v3 (当前) | Electron | `%APPDATA%\filepilot` |

### 卸载脚本不完整

原卸载脚本只清理了 2 个位置：
- `%APPDATA%\filepilot` ✅
- `%USERPROFILE%\.filepilot` ✅

遗漏了：
- `%APPDATA%\fyne\com.filepilot.app` ❌
- 桌面快捷方式 ❌
- 开始菜单 ❌
- 最近文档记录 ❌

---

## 修复方案

### 1. 更新卸载脚本 (`installer.nsh`)

```nsis
!macro customUnInstall
  MessageBox MB_YESNO "是否删除所有数据？" IDYES deleteData IDNO keepData

  deleteData:
    ; Electron 版本数据
    RMDir /r "$APPDATA\filepilot"
    
    ; 旧版 Go 程序数据
    RMDir /r "$PROFILE\.filepilot"
    
    ; 旧版 Fyne 程序数据
    RMDir /r "$APPDATA\fyne\com.filepilot.app"
    
    ; 快捷方式
    Delete "$DESKTOP\FilePilot.lnk"
    RMDir /r "$SMPROGRAMS\FilePilot"
    
    ; 最近文档记录
    Delete "$APPDATA\Microsoft\Windows\Recent\FilePilot.lnk"

  keepData:
  done:
!macroend
```

### 2. 创建清理检查清单规则

文件：`.claude/rules/cleanup-checklist.md`

要求：
- 每次添加新的数据存储位置时，必须同步更新清理脚本
- 构建前验证清理脚本完整性
- 测试时验证卸载是否彻底

---

## 经验教训

1. **数据位置必须统一管理**：不能随意选择存储位置，需要有文档记录
2. **卸载逻辑必须与安装逻辑同步**：安装时创建什么，卸载时就要清理什么
3. **技术栈迁移要考虑数据兼容**：旧版本数据位置需要纳入清理范围
4. **测试卸载流程**：不能只测试安装，卸载同样重要

---

## 验证方法

```bash
# 1. 安装新版本
FilePilot Setup 1.3.1.exe

# 2. 添加文件夹，建立索引

# 3. 卸载（选择删除数据）

# 4. 检查以下目录是否还存在
dir %APPDATA%\filepilot
dir %USERPROFILE%\.filepilot
dir %APPDATA%\fyne\com.filepilot.app

# 5. 重新安装，验证是否为全新状态
```
