# 本地文件夹文件索引与预览系统 — 开发需求文档 v2.2

> **版本更新说明**：
> 1. 去除 Windows 系统级右键菜单注册表相关描述，仅保留程序内右键菜单
> 2. 新增浏览器式交互逻辑：搜索后点击文件直接在当前标签页预览，覆盖搜索界面
> 3. 新增前端技术选型灵活性：如 Fyne 体验不佳，可使用 Go 做后端 + H5 做前端 UI
> 4. 明确 Material Design 3 (Material You) 设计规范要求

## 一、技术选型硬性规定

|模块|选型|理由|
|---|---|---|
|编程语言|Go 1.21+|跨平台、单文件部署|
|GUI 框架|**方案A**: Fyne v2（纯 Go）<br>**方案B**: Go 后端 + Webview/H5 前端|方案A 无需额外依赖；方案B 界面更美观，支持 MD3|
|数据库|SQLite（驱动：`github.com/glebarez/sqlite`）|纯 Go 实现，无需安装，单文件数据库|
|Word 转 MD|`github.com/chenjunqian/go-markitdown`|支持 DOCX 转 Markdown|
|PDF/Excel 预览|内置库 + `go-ole`（COM 调用）|提取文本和表格内容|
|打包工具|Inno Setup 或 NSIS|生成标准安装包，支持桌面图标|

### 技术选型说明

**方案A (Fyne v2)**：
- 优点：纯 Go，无 CGO 依赖，单文件部署
- 缺点：UI 组件有限，难以实现 Material Design 3 风格

**方案B (Go + H5) - 推荐**：
- 优点：UI 完全可控，可使用 CSS 实现 Material Design 3，支持动画和丰富交互
- 缺点：需要 Webview 运行时，打包体积略大
- 推荐库：
  - `github.com/webview/webview` - 轻量级 Webview
  - 前端：React/Vue + Material Design 3 组件库（如 MUI、Vuetify）

## 二、全局交互与数据规范

### 2.1 数据库设计（重要）

**禁止**为每个文件夹创建独立表。使用**单表**结构，字段如下：

```sql
CREATE TABLE files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    root_path TEXT NOT NULL,          -- 用户添加的根目录绝对路径
    relative_path TEXT NOT NULL,      -- 相对于 root_path 的路径
    file_name TEXT NOT NULL,
    file_size INTEGER,                -- 字节
    modified_time INTEGER,            -- Unix 时间戳，用于增量更新
    file_ext TEXT,
    is_indexed BOOLEAN DEFAULT 0,     -- 是否已提取文本内容用于全文搜索
    UNIQUE(root_path, relative_path)
);
-- 全文搜索虚拟表（使用 FTS5）
CREATE VIRTUAL TABLE files_fts USING fts5(content, tokenize='porter');
```

### 2.2 增量更新策略

- 每次启动软件时，对已添加的根目录执行增量扫描：
  - 比对 `modified_time`，仅更新发生变化的文件
  - 新增文件插入，删除文件（路径不再存在）从数据库中移除
- 用户可手动点击"强制全量重建索引"

### 2.3 文件内容索引（搜索优化）

- **遍历时**：只提取并存储文件名、路径、大小、修改时间等元数据，不提取文件内容，以保证入库速度
- **首次搜索时**：若 `is_indexed = 0`，则异步调用转换库提取该文件的纯文本内容，写入 FTS5 虚拟表，并将 `is_indexed` 置为 1
- **后续搜索**：直接查询 FTS5 表，毫秒级返回

### 2.4 标签页规则

- 同一文件双击多次，只激活已有标签页（不重复新建）
- 标签页标题 = 文件名，鼠标悬停显示完整绝对路径
- 关闭标签页时释放该文件占用的内存资源（如有缓存）

### 2.5 程序内右键菜单（无需注册表）

在以下界面中，为列表项绑定右键菜单：

- 首页搜索结果列表
- 首页最近打开记录列表
- 文件夹管理页面的卡片（可右键移除或重新索引）

**右键菜单统一选项**：
- 打开（预览）
- 打开所在位置（用系统资源管理器打开文件夹并选中该文件）
- 复制完整路径
- （如适用）从记录中移除或删除索引

## 三、界面布局与设计规范（UI 风格）

### 3.0 Material Design 3 (Material You) 设计规范

**严格遵循 Material Design 3 设计规范**，包括：

- **颜色系统**：Primary, Secondary, Surface, Background, Error 等完整色板
- **圆角**：所有卡片、按钮、输入框使用 12px 圆角
- **阴影层级**：Elevation 0-3 级，卡片默认 Level 1
- **字体层次**：Display, Headline, Title, Label, Body 五级字体
- **间距系统**：4px, 8px, 16px, 24px, 32px 标准间距
- **按钮样式**：Filled, Outlined, Text, Elevated, Tonal 五种按钮
- **组件样式**：Card, Chip, FAB, NavigationBar, Dialog 等标准组件

### 3.1 整体布局

- 左侧边栏：宽度 220px，背景色 `#1e1e2f`（深色）/ `#f5f5f7`（浅色）
- 右侧主区域：展示内容（首页、文件夹管理、设置、统计）
- 顶部无标题栏（Fyne 默认无边框），窗口大小默认 1280x720，可缩放

### 3.2 侧边栏菜单

固定四个入口（图标+文字）：
1. 首页
2. 文件夹
3. 设置
4. 统计

### 3.3 主题切换

- 支持"浅色/深色/跟随系统"三种模式
- 所有组件颜色需跟随主题变化

### 3.4 卡片规范

- 卡片圆角 12px，间距 16px
- "添加文件夹"卡片永远排在第一位，采用虚线边框和加号图标，强调色 `#4f8cf7`
- 已添加的文件夹卡片显示：文件夹名称、文件总数、总大小（自动换算 KB/MB/GB）

### 3.5 进度动画

- 入库时，屏幕中央弹出模态框，背景带毛玻璃遮罩
- 进度使用环形进度条（显示百分比），并附文字说明
- 模态框包含"暂停"和"取消"按钮（取消时回滚本次操作）

## 四、功能模块详细说明

### 4.1 首页（搜索 + 最近记录）- 浏览器式交互

**重要：首页采用浏览器式交互逻辑**

1. **初始状态**：
   - 搜索框居中显示（类似 Google 首页）
   - 下方显示最近打开记录列表
   - 未添加文件夹时，搜索框禁用并显示提示

2. **搜索流程**：
   - 用户输入关键词，点击搜索或按回车
   - 搜索结果显示在当前页面（覆盖初始状态）
   - 结果列表显示：文件名、所在根目录、文件大小、修改时间

3. **点击文件预览**：
   - 点击搜索结果中的文件，直接在当前标签页打开预览
   - 预览内容覆盖搜索结果界面（类似浏览器点击链接）
   - 用户可通过返回按钮回到搜索结果

4. **新标签页行为**：
   - 点击侧边栏"首页"或使用快捷键新建标签页
   - 新标签页显示首页初始状态（搜索框 + 最近记录）
   - 每个标签页独立维护自己的浏览历史

5. **最近打开记录**：
   - 按访问时间倒序，最多 50 条
   - 每条显示文件名和所在文件夹名（长路径用省略号截断中间部分）
   - 点击直接预览

### 4.2 文件夹（索引管理）

#### 交互流程

1. 点击"添加文件夹"卡片 -> 弹出系统文件夹选择对话框
2. 用户选择目录后，立即执行预扫描（快速统计）：
   - 统计文件总数、总大小（需递归遍历所有子目录）
   - 基于文件数 x 平均处理耗时 2ms，估算总索引时间并弹窗确认
3. 用户确认后开始正式索引（后台协程）：
   - 遍历所有子目录，忽略系统隐藏文件夹（以 `.` 开头）和大于 100MB 的大文件
   - 进度模态框显示，并禁止所有菜单操作
   - 索引完毕后关闭模态框，刷新卡片列表
4. 卡片列表展示所有已添加的根目录

#### 并发与资源控制

- 索引任务使用 goroutine，限制最大并发数（如 CPU 核心数 x 2），避免 IO 过载

### 4.3 设置

- **主题选择**：单选按钮（浅色/深色/跟随系统）
- **索引阈值**：数字输入框（默认 100），单位 MB，用于跳过过大的文件
- **是否索引隐藏文件夹**：开关（默认关闭）
- **开机自启动**：开关（写入注册表）
- **重置所有数据**：危险按钮，需二次确认

### 4.4 统计（图表）

- 饼图展示文件数量占比（按扩展名分组），后缀未知的归为"未知"
- 饼图下方附带表格，显示：扩展名 | 文件数量 | 占用空间（MB） | 占比
- 点击饼图的某一块，跳转到首页并自动填充搜索框为该扩展名

## 五、预览支持详细列表

|格式|预览方式|复制内容支持|
|---|---|---|
|.docx|转换为 Markdown 预览（尽量保留格式）|复制渲染后的文本|
|.xlsx|转换为 HTML 表格预览|复制表格内容（CSV格式）|
|.pptx|提取文本+图片（图片缩略）|复制文本内容|
|.pdf|使用内置渲染器显示（或转图片）|复制选中文本（若支持）|
|.txt, .md|直接文本显示（支持语法高亮）|复制全文|
|.json, .xml, .csv|格式化的树形/表格视图|复制原数据|
|其他|显示"不支持预览"，仅提供"打开所在位置"和"复制路径"|无|

## 六、打包与安装规范

### 6.1 编译命令（生产环境）

```cmd
set GOOS=windows
set GOARCH=amd64
go build -ldflags="-H windowsgui -s -w" -o myapp.exe
```

- `-H windowsgui`：隐藏控制台窗口
- `-s -w`：去除调试信息，减小体积

### 6.2 安装包制作（Inno Setup 脚本要点）

- 安装目录：`{pf}\MyFileSearcher`
- 创建桌面快捷方式
- 提供卸载程序，卸载时询问是否删除数据库文件

### 6.3 首次启动

- 若数据库不存在，自动创建并初始化表结构
- 若未添加任何文件夹，首页显示引导提示

## 七、非功能性需求

- **性能**：5000 个文件入库时间 <= 5 分钟（SSD 环境）
- **内存**：运行时内存占用 <= 200MB（预览大文件除外）
- **稳定性**：索引过程异常退出不应损坏数据库（使用事务）
- **日志**：写入 `logs/app.log`，记录错误和关键操作，便于排查

## 八、开发里程碑（建议）

1. 基础框架：Fyne 界面 + SQLite 初始化
2. 文件夹遍历与元数据存储（含增量更新）
3. 搜索功能（基于文件名，后续集成 FTS5）
4. 预览标签页（先支持 TXT 和 MD）
5. 集成 Office 转换库，扩展预览格式
6. 统计图表
7. 设置与主题切换
8. 程序内右键菜单实现
9. 打包与安装测试

## 九、补充说明

- 本项目为桌面单机应用，不涉及网络通信
- 所有第三方库应优先选择纯 Go、无 CGO 依赖的版本，以降低编译和部署复杂度
- 如 Fyne 无法满足 UI 需求，可切换到 Go + H5 方案（使用 Webview）
